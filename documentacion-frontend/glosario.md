# Glosario

* **REST (Representational State Transfer):** Es un estilo arquitectónico o conjunto de reglas y restricciones de diseño para crear servicios web. Define que la comunicación entre un cliente y un servidor debe ser **sin estado** (stateless), usar una **interfaz uniforme** y estar **orientada a recursos** (datos identificables por una URL). Es, en esencia, el "manual de reglas".
* **RESTful:** Es el adjetivo que se le aplica a un servicio web, API o sistema que implementa y cumple con los principios de REST. Si una API sigue las reglas de REST, se dice que es una **"API RESTful"**. Es la aplicación práctica del concepto.
* **API (Application Programming Interface):** Es el "puente" o intermediario que permite que dos aplicaciones se comuniquen y compartan datos entre sí de forma estandarizada.
